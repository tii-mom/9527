# 14. 还需要讨论和补充的问题

## 1. 账号系统

需要决定：

```txt
匿名模式是否允许开始赚积分？
兑换前是否必须绑定邮箱/GitHub？
是否支持 GitHub OAuth？
```

建议：允许匿名开始，兑换前必须绑定邮箱或 GitHub，提现前必须绑定钱包。

## 2. 钱包

需要决定：

```txt
是否 V1 就支持钱包绑定？
支持 EVM 还是 Solana？
USDT/USDC 支持哪些链？
```

建议：V1 不强制钱包，V2 支持 EVM 钱包签名，USDC 优先，USDT 后置。

## 3. 积分命名

候选：

```txt
9527 Points
Dev Points
Fa Points
Build Points
AI Credits
```

建议：产品未定前用 Points，正式上线前再定品牌化名字。

## 4. 积分兑换比例

需要决定是否公开参考美元价值、是否设置动态兑换比例。建议商城显示实时兑换价，不承诺永久固定美元价值。

## 5. 第一个 sponsor

需要确定第一批广告主是谁。建议先 mock 3 条开发者广告，然后找 3-5 个真实 sponsor 测试。

## 6. 第一批商城商品

需要确定模型 token 来源、云服务 credit 合作方、USDT/USDC 是否只是 waitlist。建议 V1 先用兑换码/人工发放，USDT/USDC 做 waitlist。

## 7. 终端广告频率

建议：

```txt
每小时最多 5 次
每天最多 30 次
同一命令 session 最多 1-2 次
```

## 8. 是否做邀请

建议 V1 不做，V1.1 再做。邀请奖励条件：被邀请用户完成首次广告、被邀请用户赚满 1000 points 后，邀请人才获得奖励。

## 9. 是否做排行榜

建议后置。排行榜会刺激传播，也会刺激刷量。

## 10. 是否做 VS Code 插件

建议 V2 后置。先验证 CLI。

## 11. 是否接 Carbon Ads

需要讨论。可以作为广告供应参考，但如果涉及用户积分奖励，需要确认广告网络政策。V1 最稳是自营 sponsor。

## 12. 法律文本

上线前必须补：

```txt
用户协议
隐私政策
积分规则
商城兑换规则
广告政策
反作弊政策
Crypto 兑换限制
```
