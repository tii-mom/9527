# 11. 合规与风控

## 核心原则

```txt
用户主动安装
用户主动运行
用户主动 opt-in
广告明确标注
不偷偷展示
不在 postinstall 展示广告
不默认直接发币
不承诺投资收益
提现后置风控
```

## 为什么 V1 不直接发币

如果产品表达为“看广告直接赚 USDT/代币”，容易引发：

```txt
广告平台政策风险
金融合规风险
薅羊毛流量
开发者社区反感
广告主不信任
提现欺诈
```

因此 V1 应该是：

```txt
广告奖励 -> 平台积分 -> 积分商城 -> 合规兑换
```

## npm 风险

不要做：

```txt
postinstall 广告
依赖包偷偷展示广告
自动打开浏览器
强制广告
伪装成普通工具
```

必须做：

```txt
README 透明说明
用户主动安装
用户主动运行 wrapper
广告可关闭
CI 禁用
```

## 隐私原则

不上传：源码、.env、SSH key、API key、文件内容、完整目录结构。

可上传：命令类型、包管理器、语言类型、操作系统、CLI 版本、粗略地区。

## V1 限频

```txt
单用户每小时最多 5 次奖励
单用户每天最多 30 次奖励
单设备每小时最多 5 次奖励
单 IP 每小时最多 20 次奖励
同一广告每天最多奖励 1 次
新账号前 24 小时最多 500 pending points
```

## 高风险行为

```txt
同一 IP 大量账号
同一钱包大量账号
广告完成时间过短
session 重放
伪造 API 请求
只点击不注册
数据中心代理
自动化脚本
异常高完成率
```

## 处置方式

```txt
降低奖励
积分 pending 延长
积分 locked
禁止提现
要求邮箱/GitHub 验证
要求钱包签名验证
人工审核
封禁账号
撤销积分
```

## Crypto 兑换限制

USDT/USDC/代币兑换需要：

```txt
最低积分门槛
钱包验证
支持地区判断
高风险账号审核
必要时 KYC
每周批量处理
```

## 需要准备的法律文本

```txt
用户协议
隐私政策
积分规则
商城兑换规则
广告内容政策
反作弊规则
Crypto 兑换地区限制
风险提示
```

## 风险提示文案

```txt
9527 Points 是平台奖励积分，不是证券、存款、货币或投资产品。
积分兑换比例以商城实时显示为准。
Crypto 兑换功能仅在支持地区、通过风控审核后开放。
平台有权对作弊、异常或违规账号冻结、撤销积分或拒绝兑换。
```
